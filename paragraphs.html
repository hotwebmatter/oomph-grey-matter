When we build with Drupal, we like to spec out a way to build flexible content. This is an attempt to document that approach and make it consistent from project to project.

Content Components

Content components are agnostic of layout and are the basis for any page. Without content, you got nothing.

Body Content

WYSIWYG content, including images, tables, and buttons. You know, the works.

Image Content

A media element using Drupal Responsive Images and Focal Point modules.

Video Content

Drupal will store YouTube or Vimeo identifiers in its own content type. An author need only choose the video that they want to place in a paragraph and the template does all the work of creating the needed embed code.

Gallery Content

If we can have images and videos, than why not a collection of images and videos?

Testimonials/Quotes

It might be a good idea for some sites to handle quotes in a consistent way. Fields should include quote body (maybe multiple lengths?), quote citation (optional), quote citation description (for a job title when needed, optional), and a quote title.

Layout

While Bootstrap has a large variety of possible layouts with its grid structure, we want to give authors a limited set of the most used grid patterns to choose from.
